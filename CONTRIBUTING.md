# Contributing to MASA Framework

Thank you for your interest in contributing to MASA! Contributions, critiques, and empirical evaluations are welcome. Open a discussion, submit a pull request, or reach out directly.

## How to Contribute

### Reporting Issues
- Use [GitHub Issues](https://github.com/nicolasfmelo/masa-framework/issues) for bugs, questions, or suggestions.
- Include as much context as possible: steps to reproduce, expected vs. actual behavior, environment details.

### Submitting Changes
1. Fork the repository.
2. Create a descriptive branch (`feature/add-rust-guide`, `fix/linter-false-positive`).
3. Make your changes with clear, atomic commits.
4. Open a Pull Request against `main` with a concise description of **what** and **why**.

### What We're Looking For
- **Empirical evaluations** — Run the experiments on new models, languages, or domains and share your results.
- **Language guides** — Add implementation references for languages not yet covered (Rust, Java, C#, etc.).
- **Linter improvements** — Expand the automated violation detection with new checks or reduced false positives.
- **Specification refinements** — Propose clarifications, edge-case coverage, or improved examples for the five rulesets.
- **Case studies** — Contribute new experiment cases at different complexity tiers.

### Style Guidelines
- All documentation and code comments must be in **English**.
- Follow existing patterns in the codebase (naming conventions, directory structure).
- Keep PRs focused — one logical change per PR.

### Running Experiments
See [`experiments/README.md`](experiments/README.md) for setup and reproduction instructions.

## Code of Conduct
Be respectful, constructive, and inclusive. We're all here to learn and improve how AI agents interact with software architecture.
